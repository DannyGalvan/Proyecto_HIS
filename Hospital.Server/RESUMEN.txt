PATRONES IMPLEMENTADOS
================================================================================

1. FACTORY PATTERN
   → DatabaseConfiguratorFactory selecciona el configurador correcto

2. STRATEGY PATTERN
   → IDatabaseConfigurator define la interfaz
   → Cada proveedor implementa su estrategia

3. DEPENDENCY INJECTION
   → Extensión DatabaseServiceExtensions registra servicios
   → Toda la configuración es inyectable

4. OPTIONS PATTERN
   → DatabaseSettings lee desde appsettings.json
   → IOptions<DatabaseSettings> inyectable en servicios

================================================================================
COMO USAR (EN 3 PASOS)
================================================================================

PASO 1: Editar appsettings.json
────────────────────────────────

{
  "Database": {
    "Provider": "PostgreSql",  ← CAMBIAR AQUÍ
    "ConnectionString": "Host=localhost;Port=5432;Database=proyecto_db;Username=postgres;Password=postgres;",
    "EnableAutoMigration": true
  }
}

PASO 2: Ejecutar migración
────────────────────────────

$ dotnet ef database update

PASO 3: ¡Listo!
────────────────────────────

Tu aplicación usa PostgreSQL sin cambios en el código.
